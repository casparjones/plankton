# Log 2026-03-11

## Session: Drag & Drop, Batch-Move API, Markdown, UX Fixes

### Erledigt

**Task Drag & Drop (Fix)**
- `@change`-Handler war defekt: `makeChangeHandler(col.id)` gab eine Funktion zurück die von Vue verworfen wurde — `onTaskChange` wurde nie aufgerufen
- Umgestellt auf `@update` und `@add` Events (native SortableJS-Events)
- `moveTask()` aus project-service ersetzte `state.project` → Watcher feuerte `refreshColumnTasks()` → VueDraggable-State wurde resettet → Task sprang zurück
- Fix: API direkt aufrufen (`api.post`) ohne `state.project` zu überschreiben

**Batch-Move API**
- Neuer Endpoint `POST /api/projects/:id/tasks/batch-move`
- Request: `{ moves: [{ task_id, column_id, order }] }`
- Ein Request, ein DB-Write — keine CouchDB Revision-Konflikte mehr
- Backend: `BatchMoveRequest` / `BatchMoveItem` Types in `requests.rs`
- Frontend: `persistMoves()` sendet alle betroffenen Task-Positionen in einem Call
- Nur Tasks im betroffenen Bereich (oldIndex..newIndex) werden aktualisiert

**Column Drag & Drop**
- `Sortable.create()` direkt auf dem Board-Container
- `handle: '.kanban-board-header'` — nur am Spalten-Header greifbar
- `draggable: '.kanban-column:not(.col-done)'` — Done-Spalte fixiert
- Reihenfolge wird aus DOM gelesen und via `api.put` ans Projekt gespeichert

**Scrollbare Spalten**
- `.kanban-column` bekommt `max-height: calc(100vh - 140px)`
- `.kanban-drag` bekommt `min-height: 0; overflow-y: auto`
- Kein `overflow: hidden` auf der Column — das hatte SortableJS blockiert

**Markdown-Rendering**
- `marked` Library installiert (v-html Rendering)
- Task-Beschreibung und Kommentare werden als Markdown gerendert
- CSS-Styles für Headlines, Code-Blöcke, Listen, Tabellen, Blockquotes etc.

**Login-Flash behoben**
- Neues `authChecked` ref — Login-Seite wird erst nach Auth-Check (`/auth/me`) angezeigt
- Kein kurzes Aufblitzen der Login-Seite mehr bei eingeloggten Usern

**Weitere Fixes**

- `autocomplete="one-time-code"` auf Projektname-Inputs

### Learnings
- VueDraggable (`vue-draggable-plus`): `@change` ist unzuverlässig, `@update`/`@add` sind die richtigen Events
- `v-model` auf VueDraggable steuert den visuellen State — Backend-Updates dürfen `state.project` nicht ersetzen
- CouchDB: parallele Writes auf dasselbe Dokument → Revision-Konflikte → Batch-Endpoint nötig
- `overflow: hidden` auf einem Parent-Element blockiert SortableJS Drag & Drop
- `min-height: 0` auf Flex-Children ist nötig damit `overflow-y: auto` greift
